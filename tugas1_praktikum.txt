TUGAS 1
1.	Jelaskan apa yang dimaksud dengan dokumen root?
Jawaban : Dokumen root itu seperti "induk" atau "pusat" dari semua dokumen lain dalam sebuah sistem. Bisa berupa folder utama di komputer atau file utama di website. Intinya, semua data atau file lain biasanya berhubungan atau berasal dari dokumen root ini.

2.	Pada apache web server, apakah dokumen root harus terletak dalam direktori C:\xampp\htdocs ? Jelaskan jawaban anda
Jawaban : Tidak harus, tapi default-nya memang di C:\xampp\htdocs. Jika ingin mengubahnya, cukup ganti path tersebut ke lokasi lain, Setelah itu, restart Apache agar perubahan berlaku. Jadi, document root tidak harus berada di C:\xampp\htdocs, tetapi bisa dipindahkan ke lokasi lain dengan konfigurasi yang sesuai.
3.	Jelaskan apa yang dimaksud dengan perintah git : init, clone, remote, add, commit, push dan checkout
Jawaban:   -  init: Membuat repository baru
-	Clone: salin repository dari server ke computer local
-	Remote: menghubungkan repository local denngan repository di server
-	Add: menambahkan file ke staging area
-	Commit: menyimpan perubahan secara local dengan pesan tertentu
-	Push: mengirim perubahan dari local ke repository di server
-	Checkout: berpindah ke branch lain atau versi tertentu dari kode.

4.	Jelaskan perbedaan antara GitHub dan GitLab 
Jawaban: GitHub lebih popular, gampang dipakai, dan cocok buar open-source. Sedangkan GitLab punya fitur CI/CD bawaan dan bisa di host di server sendiri, jadi lebih fleksibel.
5.	Apa bedanya variable user dengan variable system ? 
Jawaban: Variable user hanya berlaku buat satu pengguna, sementara variable system berlaku buat semua pengguna di komputer.

6.	Apa persamaan variable system dengan variable constant ?
Jawaban: Keduanya menyimpan nilai buat sistem atau program. Bedanya, variable system bisa diubah, sedangkan variable constant tidak bisa diubah.
